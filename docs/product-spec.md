# MasterCoder - AI 编程客户端产品文档

## 1. 产品概述

MasterCoder 是一款基于终端的 AI 编程助手客户端，通过接入 OpenAI 兼容接口与大语言模型交互，帮助开发者在命令行环境中完成代码编写、调试、重构等软件开发任务。

---

## 2. 核心交互模式

### 2.1 对话式交互

- 用户在终端中以自然语言描述需求，AI 返回回答或执行操作
- 支持多轮上下文对话，AI 能理解前序对话内容
- 支持流式输出（Streaming），逐 token 实时显示 AI 回复

### 2.2 操作确认机制

- AI 执行文件写入、命令运行等操作前，需向用户展示操作内容并请求确认
- 用户可选择：同意 / 拒绝 / 修改后执行
- 支持"自动批准"模式，跳过确认直接执行（适用于信任场景）

---

## 3. 功能模块

### 3.1 文件操作

| 功能 | 说明 |
|------|------|
| 读取文件 | 读取指定路径的文件内容，作为上下文提供给 AI |
| 创建文件 | 根据 AI 生成的内容创建新文件 |
| 编辑文件 | 对已有文件进行精确的局部修改（基于搜索替换或 diff） |
| 文件搜索 | 按文件名模式（glob）查找项目中的文件 |
| 内容搜索 | 按关键词或正则表达式搜索文件内容（类似 grep） |

### 3.2 命令执行

| 功能 | 说明 |
|------|------|
| 执行 Shell 命令 | AI 可调用 shell 命令完成构建、测试、安装依赖等操作 |
| 命令输出捕获 | 将命令的 stdout/stderr 捕获并反馈给 AI，用于后续决策 |
| 超时控制 | 命令执行设有超时限制，防止长时间阻塞 |
| 工作目录感知 | 命令在用户当前项目目录下执行，保持路径一致 |

### 3.3 上下文管理

| 功能 | 说明 |
|------|------|
| 项目目录感知 | 自动识别当前工作目录作为项目根目录 |
| Git 仓库感知 | 检测当前目录是否为 Git 仓库，获取分支、状态等信息 |
| 手动添加上下文 | 用户可通过命令主动将文件或目录加入对话上下文 |
| 上下文窗口管理 | 当对话超出模型上下文窗口时，自动压缩/截断早期消息 |

### 3.4 模型配置

| 功能 | 说明 |
|------|------|
| OpenAI 兼容接口 | 支持任何兼容 OpenAI Chat Completions API 的服务端点 |
| 自定义 API 地址 | 用户可配置 base URL，对接不同模型提供商 |
| 自定义 API Key | 用户配置自己的 API Key 进行鉴权 |
| 模型选择 | 用户可指定使用的模型名称（如 gpt-4o、deepseek-chat 等） |
| 多模型配置 | 支持保存多组模型配置，运行时切换 |

## 4. 工具系统（Tool Use）

AI 通过调用预定义的工具来与本地环境交互。以下为内置工具：

### 4.1 工具列表

| 工具名称 | 功能描述 |
|----------|----------|
| `read_file` | 读取指定文件内容 |
| `write_file` | 创建或覆写文件 |
| `edit_file` | 对文件进行局部编辑（搜索替换） |
| `list_files` | 列出目录下的文件（支持 glob 模式） |
| `search_files` | 在文件内容中搜索关键词/正则 |
| `run_command` | 执行 shell 命令并返回输出 |

### 4.2 工具调用流程

1. AI 根据用户需求决定调用哪个工具及参数
2. 客户端拦截工具调用请求，展示给用户
3. 用户确认后，客户端在本地执行工具操作
4. 将执行结果返回给 AI，AI 继续推理

## 5. 会话管理

### 5.1 对话生命周期

| 功能 | 说明 |
|------|------|
| 新建会话 | 启动时默认创建新的对话会话 |
| 会话持续 | 在同一终端会话中保持对话上下文连续 |
| 清空会话 | 用户可手动清空当前对话历史，重新开始 |
| 会话恢复 | 支持恢复上次未完成的会话（可选） |

### 5.2 对话历史

| 功能 | 说明 |
|------|------|
| 本地持久化 | 对话历史保存在本地文件系统 |
| 历史浏览 | 用户可查看过去的对话记录 |

---

## 6. 终端界面（TUI）

### 6.1 输入区域

- 多行文本输入，支持换行编辑
- 输入历史记录，可通过上下键浏览之前的输入
- 支持粘贴多行文本

### 6.2 输出区域

- Markdown 渲染：AI 回复中的代码块、列表、标题等正确渲染
- 代码高亮：代码块按语言进行语法高亮
- Diff 展示：文件修改以 diff 格式直观展示变更内容
- 流式输出：AI 回复逐步显示，而非等待全部完成

### 6.3 状态指示

- 显示当前使用的模型名称
- 显示 AI 正在思考/执行工具的状态
- 显示 Token 用量（输入/输出 token 数）
- 显示当前工作目录和 Git 分支

---

## 7. 斜杠命令（Slash Commands）

用户可通过 `/` 前缀的快捷命令执行常用操作：

| 命令 | 功能 |
|------|------|
| `/help` | 显示帮助信息 |
| `/clear` | 清空当前对话 |
| `/model` | 切换模型 |
| `/exit` | 退出程序 |
| `/config` | 打开/编辑配置 |

---

## 8. 配置系统

### 8.1 配置文件

- 全局配置文件：存放在用户 home 目录下（如 `~/.mastercoder/config.json`）
- 项目级配置文件：存放在项目根目录下（如 `.mastercoder/config.json`），可覆盖全局配置

### 8.2 可配置项

| 配置项 | 说明 |
|--------|------|
| `api_base_url` | API 服务端点地址 |
| `api_key` | API 鉴权密钥 |
| `model` | 默认使用的模型名称 |
| `max_tokens` | 单次回复最大 token 数 |
| `temperature` | 模型生成温度参数 |
| `auto_approve` | 是否自动批准工具调用 |
| `system_prompt` | 自定义系统提示词（可选） |

### 8.3 环境变量支持

- 支持通过环境变量设置 API Key 等敏感配置（如 `MASTERCODER_API_KEY`）
- 环境变量优先级高于配置文件

---

## 9. 系统提示词（System Prompt）

### 9.1 内置系统提示词

- 客户端内置一套默认系统提示词，定义 AI 的角色和行为规范
- 包含：角色定义、工具使用说明、安全约束、输出格式要求

### 9.2 自定义系统提示词

- 用户可通过配置文件自定义系统提示词
- 支持项目级 prompt 文件（如 `.mastercoder/prompt.md`），自动注入项目特定指令

---

## 10. 安全与权限

### 10.1 操作审批

- 所有文件写入、命令执行默认需要用户确认
- 敏感操作（如删除文件、`rm -rf`、`git push --force`）需特别提示

### 10.2 沙箱约束

- 文件操作限制在项目目录范围内（可配置是否允许访问项目外路径）
- 可配置命令黑名单，禁止执行特定危险命令

### 10.3 密钥安全

- API Key 不在终端界面中明文展示
- API Key 不写入对话历史或日志

---

## 11. 项目指令文件

### 11.1 MASTERCODER.md

- 项目根目录下可放置 `MASTERCODER.md` 文件
- 文件内容会自动加载为项目级指令，注入到对话上下文中
- 用途：定义项目编码规范、技术栈说明、特殊约束等
- 类似于 Claude Code 的 `CLAUDE.md` 机制

---

## 12. 错误处理

| 场景 | 处理方式 |
|------|----------|
| API 请求失败 | 显示错误信息，提示用户检查网络/配置，支持重试 |
| API Key 无效 | 明确提示鉴权失败，引导用户检查 Key 配置 |
| 模型不存在 | 提示模型名称无效，列出可用模型（如 API 支持） |
| 文件操作失败 | 显示具体错误（权限不足、路径不存在等） |
| 命令执行超时 | 终止命令并通知 AI 和用户 |
| 上下文超长 | 自动压缩历史消息，提示用户 |
